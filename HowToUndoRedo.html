How to enable Undo/Redo for Functions
WIP
When desiging a page or application, steps can be undone and redone by pressing Ctrl+z or Ctrl+y while focusung the World. This is realized by logging each morphic action (like Position setting or Morph addition) in a resource called GlobalLogger. For your extensions of Lively Kernel, you can enable logging for you custom functions, too. This is required, whenever you enable a functionality that does not access basic morph functions, which can be setter (setPosition), addition and removal of morphs, stepping of scripts or enabling and disabling of properties of morphs.
To make 'switchPlayer' loggable, define the Function 'logSwitchPlayer' to lively.morphic.GameBoard. This function returns an action object that has the properties morph, undo and redo: - morph is the target of the undo action, in this case the Game - undo is a function that will be applied to the target when hitting Ctrl+z - redo is a function that will be applied to the target when hitting Ctrl+y lively.morphic.Game.addMethods( 'logging' { logSwitchPlayer: function (/* arguments for switchPlayer */) { return { morph: this, undo: (function () { this.currentPlayerId = (this.currentPlayerId - 1 + this.playerCount) % this.playerCount this.playerChanged(); }).bind(this); redo: (function () { this.switchPlayer(); }).bind(this) } } })
Let's imagine that there is a simulation mode in our game, where actions can be tested but will not count as actual actions. You could not want to enable the users to undo such an action - in those cases, you can add a function called beforeLogSwitchPlayer, that returns false if the demo mode is turned off and true if it is turned on. beforeLogSwitchPlayer: function (/* arguments for switchPlayer */) { return !this.isInDemoMode() } Similar to the log condition, cou can add a function afterLogSwitchPlayer, that will receive the return value from switchPlayer and will be executed directly after the logging.
You are writing a class lively.morphic.GameBoard. To store the current player, a property 'currentPlayerId' is stored on a GameBoard, that can be set via the functoin 'switchPlayer'. lively.morphic.Game.addMethods( 'playing' { switchPlayer: function () { this.currentPlayerId = (this.currentPlayerId + 1) % this.playerCount this.playerChanged(); } })

How it works
By default Morphic behavior is instrumented to record morph and shape changes. The logger is enabled by the Config.undoLogging that is queried on world load in lively.Main. The GlobalLogger object (whose singleton instance you can access with $world.GlobalLogger) will then instrument Morph and Shape classes to install logging behavior. Actions that get logged are kept in a stack that can be inspected by accessing $world.GlobalLogger.stack. What gets logged is defined as follows: - There are white- and blacklisted classes and methods - For method that get logged the following policy applies: - If there are two methods starting with enable... and disable... those will be used for undo/redo behavior. - For an arbitrary methof METHODNAME the following methods if existing will be invoked when recording undo behavior: beforeLogMETHODNAME, logMETHODNAME, and afterLogMETHODNAME. See examples. The before/after methods can implement arbitrary logic. logMETHODNAME should return an object with the fields {morph: MORPH, undo: FUNCTION, redo: FUNCTION} that will be added to the stack. - Note that the items of $world.GlobalLogger.stack are lists themselves so that one undo/redo action can consist out of multiple atomic undo / redo actions.
